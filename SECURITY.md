# Security Policy

## Public repository rules

Do not commit passwords, API keys, access tokens, `.env` files, private keys, student IDs, government IDs, private client data, database backups, or production server configuration.

If a real credential is committed, rotate it immediately. Deleting the file or commit alone is not sufficient because Git history may retain it.

## Reporting a vulnerability

Use GitHub private vulnerability reporting when available. Do not publish sensitive vulnerability details in a public issue.
